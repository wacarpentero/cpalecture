SOC Engagement Reporting — Considerations

Learning Objectives

  • Explain the effect of CUECs on the SOC 1 and SOC 2 report.
  • Summarize carve‑out vs inclusive methods for reporting on CSOCs.
  • Explain types of opinions and report modifications when deficiencies are identified.
  • Prepare results of tests of controls for a SOC 2 Type 2 report, including treatment of exceptions.
  • Determine the appropriate form and content of SOC 1 / SOC 2 reports.

CUECs — How they affect the report

  • Definition: Complementary user‑entity controls that user entities must implement for the service org’s controls to achieve objectives.
  • Placement: Included in the description and referenced in control objectives/criteria as dependencies.
  • Effect on opinion: If CUECs are necessary and not implemented by user entities, user auditors must consider the impact on their audit; the service auditor’s SOC opinion is not modified solely for user entities’ failure to implement CUECs—but CUECs should be clearly presented.
  • Clarity tip: Write CUECs as explicit, testable actions for user entities (e.g., “restrict access to application via SSO and MFA”).
CUECs = user responsibility Not testing by service auditor

CSOCs — Carve‑out vs Inclusive

  • CSOCs: Controls the service organization implements to address risks from subservice organizations.
  • Carve‑out: Subservice controls excluded from testing; describe services provided and disclose relevant CSOCs in the description; may reference bridge letters.
  • Inclusive: Subservice controls included within the scope of description and testing; independence may extend to subservice org; evidence access required.
  • Reporting: The chosen method is disclosed in the description; implications flow into the testing/results section (SOC 2 Type 2).
Decision snapshot: If access to subservice evidence & independence → Inclusive If not feasible but risks managed by CSOCs → Carve‑out + disclose CSOCs

Opinions & Report Modifications

Types of opinions

  • Unmodified — description fairly presents; controls suitably designed (and operated effectively for Type 2).
  • Qualified — except for the effects of described matters (e.g., certain control objectives/criteria not met, scope limitation).
  • Adverse — material and pervasive misstatements or ineffective controls.
  • Disclaimer — auditor unable to obtain sufficient appropriate evidence.

When deficiencies are identified

  • Describe the nature of the deficiency/exception, affected objectives/criteria, and implications for achieving service commitments or ICFR.
  • Evaluate pervasiveness to decide between qualified vs adverse.
  • Reference the related results‑of‑tests table (for SOC 2 Type 2).

SOC 2 Type 2 — Results of Tests of Controls

For each control tested, present a concise, decision‑useful row:

  • Control tested and the related TSC criteria/objective.
  • Test procedure (inquiry, inspection, observation, reperformance) and period covered.
  • Sample size & selection (where relevant) and population definition.
  • Results — state no exceptions noted or detail the exceptions with frequency, cause, and impact.
  • Qualitative context — whether exceptions were isolated, systemic, or had compensating controls.

Handling exceptions

  • Explain the nature (e.g., 2 of 30 samples missing timely approval), the likely cause, and whether it prevents meeting the criterion/objective.
  • Assess whether exceptions are deviations vs. anomalies; consider projected deviation rate and precision.
  • Link to opinion: multiple or pervasive exceptions can lead to a qualified or adverse opinion.

Form & Content — SOC 1 / SOC 2

Core components of the report

  • Independent Service Auditor’s Report (opinion, basis, responsibilities, inherent limitations, restricted use).
  • Management’s Assertion (Type 1 design; Type 2 design & operating effectiveness).
  • System Description (and, for SOC 1, control objectives & related controls; for SOC 2, criteria in scope).
  • Results of Tests of Controls (SOC 2 Type 2) with exceptions described.
  • CUECs, subservice method (inclusive/carve‑out), and CSOCs disclosures.
  • Other information (if presented) clearly separated from the auditor’s report.
  • Period/point‑in‑time covered, report date, practitioner’s signature and location.

Practice Questions

1) Which statement best describes the impact of CUECs on a SOC report?

  1. CUECs must be tested by the service auditor and failures modify the SOC opinion
  2. CUECs are user responsibilities; they are disclosed, but failure to implement them by a user entity does not by itself modify the service auditor’s SOC opinion
  3. CUECs are the same as CSOCs
  4. CUECs apply only to SOC 3
Answer & Explanation

B — CUECs are responsibilities of user entities. The service auditor discloses them; user auditors consider their effect on user F/S or controls testing.

2) Under the carve‑out method, the SOC report should:

  1. Include the subservice controls and test them
  2. Exclude subservice controls from testing but describe the services and disclose CSOCs
  3. Omit any mention of the subservice organization
  4. Present CUECs as CSOCs
Answer & Explanation

B — Carve‑out excludes subservice controls from testing while describing the subservice and the CSOCs used to manage related risks.

3) A SOC 2 Type 2 “results of tests” line that reads “2 of 30 approvals untimely; deviation rate 6.7%” most likely requires the auditor to:

  1. Automatically issue an adverse opinion
  2. Evaluate frequency/cause/impact and consider whether the deviation is isolated, systemic, or offset by compensating controls
  3. Ignore the exception because it is less than 10%
  4. Reclassify it as a CUEC
Answer & Explanation

B — Exceptions are evaluated qualitatively and quantitatively; pervasiveness determines any opinion modification.

4) Which opinion is most appropriate when the auditor cannot obtain sufficient appropriate evidence?

  1. Unmodified
  2. Qualified
  3. Disclaimer
  4. Adverse
Answer & Explanation

C — Lack of evidence leads to a disclaimer of opinion.

5) Which item belongs in the form and content of a SOC 2 Type 2 report?

  1. Only management’s assertion and system description
  2. Detailed revenue testing for user entities
  3. Independent Service Auditor’s Report, management’s assertion, system description, and results of tests of controls
  4. Only the Trust Services Criteria listing
Answer & Explanation

C — Those are the core components; the SOC 2 Type 2 includes results of tests.

6) In an inclusive presentation, independence considerations may extend to:

  1. The subservice organization included in scope
  2. Only user entities
  3. The cloud provider when carve‑out is used
  4. Only the engagement quality reviewer
Answer & Explanation

A — Inclusive method brings the subservice org within scope; independence and evidence access apply.

7) Which phrase best belongs in a well‑written “results of tests” cell for an exception?

  1. “Found issues”
  2. “2/30 approvals were completed 3–5 days late; caused by staff turnover; compensating detective review mitigated risk to low”
  3. “Exception noted, N/A”
  4. “See management letter”
Answer & Explanation

B — Decision‑useful detail: frequency, timing, cause, and the effect/mitigation.

8) When deficiencies are material and pervasive to the objectives/criteria, the likely opinion is:

  1. Unmodified
  2. Qualified
  3. Adverse
  4. Disclaimer
Answer & Explanation

C — Pervasive deficiencies call for an adverse opinion.
