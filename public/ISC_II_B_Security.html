Security

Learning Objectives

  • Classify types of threat agents (internal/external, nation‑state/non‑state, adversary/attacker).
  • Identify attack types (physical, DDoS, malware/ransomware, social engineering, web & mobile app attacks).
  • Identify techniques used in cyber‑attacks (buffer overflow, mobile code, XSS, SQL injection, race conditions, covert channels, replay, return‑oriented attacks) at a high level.
  • Explain attack stages: reconnaissance → gaining access → privilege escalation → maintaining access → exploitation/exfiltration → covering tracks.
  • Identify cybersecurity risks in cloud platforms/services (misconfiguration, shared responsibility, exposed keys).
  • Identify risks for the Internet of Things (IoT) (default creds, weak update paths, insecure telemetry).
  • Identify risks for mobile technologies (device loss, rooting/jailbreak, unvetted apps, weak MDM).
  • Determine threats in connections with customers, vendors & partners and across on‑prem & cloud apps, networks, and connected devices.
Exam tip: In scenario questions, anchor on actor → asset → vector → impact. Then pick the control that reduces probability or limits impact—without giving operational “how‑to” steps.

Illustration — Attack lifecycle (recognize the stage)

1. Recon
Open‑source info, scanning.
2. Initial access
Phish, exploit vulnerable service, stolen creds.
3. Privilege escalation
Abuse misconfig, local exploit.
4. Lateral movement
Pivot to servers; spread.
5. Actions on objectives
Exfiltrate data, encrypt files, disrupt ops.
6. Cover tracks
Clear logs, disable defenses.
Exam cue: If the scenario says “public S3 bucket with customer CSVs found by researcher,” the likely control is configuration hardening & monitoring (not incident response first).

Threat modeling & landscape

How to model a threat (exam‑level)

  • Scope the asset (data/process/system) and entry points.
  • Trace actor → vector → asset → impact; sketch a simple data‑flow.
  • Apply STRIDE mnemonics: Spoofing, Tampering, Repudiation, Information disclosure, Denial of service, Elevation of privilege.
  • Pick controls that reduce likelihood (prevent/detect) or impact (limit/restore).

Current threat landscape (recognize)

  • Ransomware & data extortion via initial access brokers.
  • Business email compromise & OAuth/token abuse.
  • Supply‑chain compromises (third‑party software or access).
  • Cloud misconfiguration (public storage, over‑privileged roles).

Cloud, IoT, Mobile — focus points

On‑prem apps & networks

  • Unpatched services / legacy protocols (e.g., SMBv1, NTLM relay).
  • Flat network with shared local admin credentials.
  • Weak segmentation around crown‑jewel systems (ERP/financials).
  • Insufficient monitoring & logging for lateral movement.

Cloud checklist

  • Confirm shared responsibility for IaaS/PaaS/SaaS.
  • Harden identity: MFA, least privilege, scoped tokens, key rotation.
  • Encrypt data at rest & in transit; prevent public storage unless intended.
  • Baseline configurations; detect drift & public exposure.

IoT checklist

  • Inventory devices; change default passwords and disable unused services.
  • Isolate IoT on separate VLANs; restrict outbound traffic.
  • Enable signed updates; maintain patch cadence.

Mobile checklist

  • Deploy MDM/MAM; enforce encryption & screen lock.
  • App allow‑list; block sideloading for corporate devices.
  • Remote‑wipe & selective‑wipe for BYOD.

Testing & Reporting

Security awareness & communication

  • Obtain an understanding of how security is communicated (policies, LMS, phishing drills).
  • Assess effectiveness: participation, comprehension checks, incident trends.
  • Model expected behaviors (reporting, clean‑desk, secure transfer, password hygiene).

Assessment input

  • Document issues, evidence, impact, and recommendations.
  • Rate likelihood/impact; propose preventive/detective/corrective actions and owners.

Walkthroughs

  • Trace a transaction/control step‑by‑step; compare the observed procedure to the documented policy/standard.
  • Capture artifacts: screenshots/logs, tickets, approvals; note exceptions.

SOC 2® — Trust Services Criteria

  • Identify design deficiencies (control missing/insufficient) and operating deficiencies (not performed or ineffective).
  • Map findings to TSC categories (Security, Availability, Confidentiality, Processing Integrity, Privacy) and system commitments/requirements.

Mitigation & Controls

Using COSO to assess cyber risk

  • Map risks to COSO ERM/IC components: Control Environment, Risk Assessment, Control Activities, Information & Communication, Monitoring.
  • Evaluate design (is a control in place?) and operation (is it performed & evidenced?).
  • Document assertions, control owners, frequency, evidence, and deficiency severity.

Control types

  • Preventive: hardening/baselines, least‑privilege IAM, network segmentation, MFA, allow‑listing.
  • Detective: logging/alerting, IDS/EDR, integrity monitoring, anomaly detection.
  • Corrective: patching, isolation/quarantine, restore from backups, key rotation.

Identification & Authentication

  • Password policy & vaulting, single sign‑on, MFA (something you know/have/are).
  • PINs, digital signatures/certificates, smart cards, biometrics (consider privacy/false‑accept).
  • Scenario fit: privilege profiles, risk‑based step‑up, device trust, and session timeouts.

Authorization models

  • DAC (discretionary): owner grants rights; enforced by ACLs.
  • RBAC (role‑based): roles aggregate permissions; apply account restrictions, SoD.
  • MAC (mandatory): labels/clearances; enforced by policy & physical barriers where needed.

Practice Questions

1. Threat agent vs technique

An attacker sends convincing emails to payroll staff to capture credentials and then steals files from cloud storage. Which pair best describes the technique and the attack stage?

  1. SQL injection; exploitation
  2. Phishing; initial access
  3. Replay; reconnaissance
  4. DDoS; covering tracks
Show answer & explanation
B. Phishing is a social‑engineering technique commonly used for initial access.
2. Cloud risk identification

A team grants Administrator to a CI server so builds can publish artifacts to cloud storage. What risk is most acute?

  1. Public bucket exposure only
  2. Excessive privilege enabling broad compromise if CI is breached
  3. Lack of encryption at rest
  4. Physical theft of media
Show answer & explanation
B. Over‑privileged roles are a common cloud misconfiguration; compromise of CI would grant wide access.
3. IoT segmentation

Smart cameras were added to the corporate network and now a database server reports unusual outbound traffic. Best first control?

  1. Disable backups
  2. Place cameras on isolated network segments and restrict egress
  3. Turn off server logging
  4. Allow third‑party remote access
Show answer & explanation
B. IoT should be isolated with minimal egress; this reduces lateral movement and data exfiltration risk.
4. Mobile control

Which control most directly reduces risk of data loss if a sales rep’s phone is lost?

  1. Weekly password rotation
  2. MDM with device encryption and remote wipe
  3. Open guest Wi‑Fi access
  4. Disable screen lock
Show answer & explanation
B. MDM enforcing encryption + remote wipe addresses the asset loss scenario.
5. Partner connection threat

A vendor’s API token for your environment is found in a public repo. Which immediate action reduces blast radius?

  1. Increase log retention
  2. Rotate & scope the token to least privilege; then monitor
  3. Disable backups
  4. Whitelist all vendor IPs
Show answer & explanation
B. Rotate/limit privileges and monitor for misuse; principle of least privilege.
6. Technique classification

Untrusted input rendered in a web page executes in a user’s browser session. Which technique?

  1. Cross‑site scripting
  2. SQL injection
  3. Replay attack
  4. Buffer overflow
Show answer & explanation
A. XSS. Code runs in the victim’s browser when output encoding/input validation is missing.
